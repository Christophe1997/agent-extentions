# Chinese Blog Writing Style Guide

> **Source**: https://github.com/ruanyf/document-style-guide

This document summarizes key writing style guidelines for Chinese blog posts. For the complete guide, refer to the source repository above.

---

## 标题 (Headings)

### 层级

| 级别 | 用途 | Markdown |
|------|------|----------|
| 一级 | 文章的标题 | `# 标题` |
| 二级 | 文章主要部分的大标题 | `## 标题` |
| 三级 | 二级标题下面的小标题 | `### 标题` |
| 四级 | 三级标题下面的细分标题 | `#### 标题` |

### 原则

1. **一级标题下，不能直接出现三级标题**
2. **避免孤立编号**（同级标题只有一个）
3. **下级标题不重复上一级标题的名字**
4. **谨慎使用四级标题**，并列内容建议用项目列表：

```markdown
### 三级标题
**（1）A**
**（2）B**
**（3）C**
```

## 文本 (Text)

### 字间距

| 规则 | 示例 |
|------|------|
| 全角中文与半角英文之间加空格 | `启动 Windows 系统` |
| 英文单位与数字之间留空隙 | `16 GB` |
| 半角英文/数字与全角标点之间不加空格 | `MacBook Air。` |

### 句子

- **避免长句**：不含标点的句子 ≤20 字最佳，>40 字不可接受
- **使用简单句和并列句**，避免复合句
- **使用肯定句**，避免否定句和双重否定

### 写作风格

| ✅ 正确 | ❌ 错误 |
|--------|--------|
| 使用主动语态 | 使用被动语态 |
| 正式语言风格 | 非正式/口语化 |
| 现代汉语常用表达 | 冷僻、生造、文言文 |
| 明确代词指代 | 模糊指代 |

### 英文处理

- 英文复数翻译成中文时，还原为单数
- 首次出现英文词汇时，标注中文
- 专有名词首字母大写，非专有名词不大写

---

## 段落 (Paragraphs)

- 一个段落只能有一个主题
- 中心句子放在段首
- 段落长度 ≤7 行，最佳 ≤4 行
- 段落之间使用一个空行隔开

### 引用

- 引用第三方内容时，注明出处
- 全篇转载，注明作者和出处
- 使用外部图片，标明来源

---

## 标点符号 (Punctuation)

### 基本原则

- 中文语句使用**全角标点**
- 整句英文使用**半角标点**
- 点号不出现在标题末尾

### 常用标点

| 标点 | 用法 | 示例 |
|------|------|------|
| 顿号 `、` | 中文并列词分隔 | `Google、Facebook、腾讯` |
| 引号 `""` | 引用内容 | `"友好"和"专业"` |
| 括号 `（）` | 补充说明 | `（电缆和接插件）` |
| 冒号 `：` / `:` | 解释用全角，时间用半角 | `以下内容：` / `8:00` |
| 省略号 `⋯⋯` | 占两个汉字，六个点 | 不用 `...` 或 `。。。` |

### 注意事项

- 最后一个并列词用"和"连接
- 避免使用感叹号
- 数值范围用 `～` 或"至"

## Quick Checklist

Before publishing, verify:

- [ ] 标题层级正确（无跨级、无孤立、无重复）
- [ ] 中英文之间有空格
- [ ] 句子长度适中（无超长句）
- [ ] 使用主动语态和肯定句
- [ ] 代词指代明确
- [ ] 使用全角中文标点
- [ ] 并列词用顿号分隔
- [ ] 引用内容注明出处
